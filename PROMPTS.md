## 1. Анализ задания и выбор стека

**Промпт:**
```
Я разрабатываю кроссплатформенное мобильное приложение «BBplay — компьютерные клубы» для конкурса. 
Приложение должно позволять:
- Авторизоваться под своими учетными данными
- Пополнять баланс
- Бронировать компьютеры для посещения
- Задавать вопросы через чат-бот на основе внутренней базы знаний

Компания Black Bears занимается разработкой мобильных игр, офис в Тамбове.
Выбери оптимальный tech stack для быстрой разработки, деплоя по одной ссылке
и работы на iOS и Android без app store. Обоснуй выбор.
```

**Результат:** Выбор React PWA + Vite + Supabase + Claude API

## 6. Система бронирования

**Промпт:**
```
Разработай страницу бронирования компьютеров для приложения BBplay.
Требования:
- Сетка компьютеров разбитая по зонам: Standard / VIP / CYBER
- Цветовые статусы: свободно (cyan), занято (red), бронь (yellow)
- Фильтр по зоне (tabs)
- Легенда статусов
- При клике на свободный компьютер — bottom sheet модалка с:
  - Характеристиками ПК (CPU, GPU, RAM, монитор, Hz)
  - Выбором времени начала (select часы)
  - Выбором длительности (1-8 часов)
  - Расчётом стоимости
  - Проверкой достаточности баланса
  - Кнопкой подтверждения
- После бронирования: списание с баланса, запись в историю транзакций, toast

Моки компьютеров: 8 Standard (80₽/ч), 6 VIP (140₽/ч), 4 CYBER (200₽/ч).
```

## 10. Supabase схема базы данных

**Промпт:**
```
Разработай SQL схему для Supabase (PostgreSQL) для приложения BBplay.
Таблицы: profiles, clubs, computers, bookings, transactions.
Требования:
- Row Level Security (RLS) для всех таблиц
- Триггер auto-create profile при регистрации через auth.users
- Constraints: check overlap на bookings (нельзя дважды забронировать один ПК)
- Атомарные функции: topup_balance() и create_booking() с транзакционностью
- Seed данные для 3 клубов
- JSONB для характеристик ПК (specs)
```
